# Security Policy

## Supported Versions
- The latest minor release is actively supported for security fixes.

## Reporting a Vulnerability
- Do not open public GitHub issues for vulnerabilities.
- Report privately to project maintainers with:
  - impact summary
  - affected component and version
  - reproduction steps or proof of concept
  - suggested remediation if available

## Response Targets
- Initial triage acknowledgement: within 3 business days.
- Severity assessment and remediation plan: within 10 business days.
- Critical vulnerabilities are prioritized for out-of-band patch release.

## Security Principles
- Preserve strict read-only API behavior by default.
- Never store credentials in output artifacts.
- Enforce TLS verification unless explicitly overridden for lab troubleshooting.
